SECTION 01

실험 개요 및 배경

AI가 글을 쓸 때 다음에 올 단어를 그냥 하나 딱 고르는 게 아니라 여러 단어에 확률을 매겨서 그중 하나를 뽑는다. Temperature, Top-P, Top-K는 그 확률 분포에서 어떤 단어를 뽑을지를 조절하는 파라미터들이다. 이번 실험에서는 이 세 개를 하나씩 바꿔보면서 상위 5개 단어의 확률이 어떻게 달라지는지 측정해 봤다.

실험 설계 (단일 변수 통제법) · Temperature 통제: Top-K=0, Top-P=1.0 고정 → Temperature를 0.1 ~ 2.0으로 변화
· Top-P 통제: Temperature=1.0, Top-K=0 고정 → Top-P를 0.05 ~ 0.99로 변화
· Top-K 통제: Temperature=1.0, Top-P=1.0 고정 → Top-K를 1 ~ 500으로 변화
· 총 12개 프롬프트 × 각 파라미터별 10~11개 수준 = 360개 관측값
SECTION 02

사전 가설

실험 전에 각 파라미터가 어떻게 작동할 것 같은지 먼저 생각해봤다.

Temperature 가설
온도가 높을수록 나오는 단어의 양(다양성)이 많아질 것 같다. 온도가 낮으면 한 단어에 확률이 몰리고, 높으면 여러 단어로 골고루 퍼질 것 같다.
Top-P 가설
Top-P의 P가 '프로그램(process)'의 P인 줄 알았다. 그래서 값이 클수록 컴퓨터 리소스를 더 많이 쓸 것 같다고 생각했다.
Top-K 가설
Top-K는 가능성을 열어서 더 많은 단어를 검토하는 것 같다. K값이 커지면 후보 단어가 많아지니까 확률도 그만큼 퍼질 것 같다.
SECTION 03

파라미터 개념 정리

Temperature (온도) 확률 분포 전체를 날카롭게 만들거나 평평하게 만드는 파라미터다. 각 단어의 점수(logit)를 Temperature로 나눈 다음 softmax를 적용하는 방식으로 작동한다.

· T가 0에 가까울수록: 제일 높은 확률의 단어에 거의 100% 집중됨
· T = 1.0: 모델이 원래 학습한 분포 그대로 사용
· T > 1.0: 확률이 고르게 펴져서 다양하고 창의적인 출력이 나옴
Top-P (Nucleus Sampling) 확률이 높은 단어들을 누적해서 합이 P가 될 때까지만 남기고 나머지는 다 제외하는 방식이다. P는 'Probability(확률)'의 약자로, 리소스나 프로세스랑은 완전히 무관하다.

· Top-P = 0.05: 누적 확률 5%짜리 단어들만 허용 → 1위 단어 확률이 거의 100%
· Top-P = 1.0: 모든 단어를 후보로 허용 (필터 없음)
· 값이 클수록 후보 단어가 많아져서 확률이 분산됨
Top-K 확률 상위 K개 단어만 남기고 나머지는 전부 확률 0으로 만드는 방식이다.

· Top-K = 1: 무조건 1등 단어만 선택 → 1위 확률 100%
· Top-K = 500: 상위 500개 단어를 후보로 허용 → 확률이 넓게 분산됨
· K가 클수록 후보 단어가 많아져서 1위 단어의 확률이 상대적으로 낮아짐

SECTION 05

가설 검증 결과

🌡️ ✅
Temperature 가설 — 부분 맞음
"온도가 높을수록 단어 다양성이 늘어난다"는 맞았다. T=0.1일 때 1위 단어가 91.5%를 독점했는데 T=2.0에서는 0.9%까지 떨어져서 확률이 고르게 퍼진다. 다만 "단어의 양"보다는 확률 분포가 얼마나 평탄한지로 표현하는 게 더 정확하다.
🔵 ❌→✅
Top-P 가설 — 방향은 맞음
"리소스를 더 먹는다"는 틀렸다. P는 Probability(확률)의 약자고 리소스랑은 전혀 관계없다. 그런데 "P 값이 클수록 결과가 더 다양해진다"는 방향은 데이터로 확인됐다. P=0.05 → 92.1%, P=0.99 → 23.5%로, 낮을수록 집중, 높을수록 분산된다.
🔷 ✅
Top-K 가설 — 정확히 맞음
"K가 클수록 더 많은 단어를 검토하고 확률이 분산된다"는 완전히 맞았다. K=1에서 1위 확률 100%, K=500에서 26.5%로, K가 커질수록 후보 풀이 넓어지면서 확률이 나뉜다. 직관이 실제 작동 방식과 딱 맞은 가설이었다.

파라미터 비교 분석

6-1. 세 파라미터의 "집중도 감소 효과" 비교 (정규화, 0=최대집중 / 1=최대분산)
Fig.4 — 세 파라미터 다 값이 커질수록 확률이 분산되는 방향은 같은데, 분산되는 속도랑 방식이 서로 다르다.
파라미터 작동 방식 낮은 값 효과 높은 값 효과 핵심 특징
Temperature 확률 분포 전체를 스케일링 집중 (결정론적) 분산 (창의적) 분포 모양 자체를 변형
Top-P 누적확률 P 이하 단어만 허용 집중 (소수 단어만) 분산 (더 많은 후보) 문맥에 따라 후보 수가 달라짐
Top-K 상위 K개 단어만 허용 집중 (K=1이면 100%) 분산 (더 많은 후보) 후보 개수가 K로 항상 고정
Top-P랑 Top-K의 결정적인 차이 Top-K는 항상 K개의 단어를 후보로 고정한다. 반면 Top-P는 문맥에 따라 후보 수가 달라진다. 예를 들어 1위 단어가 이미 85% 확률이면, Top-P=0.9에서는 후보가 1~2개밖에 없지만 Top-K=10이면 무조건 10개의 단어가 후보가 된다. 그래서 실제 AI에서는 Top-P가 Top-K보다 더 자연스럽게 작동한다고 알려져 있다.
SECTION 07

AI 활용의 황금비율: 실전 레시피

실험 결과를 종합해 볼 때, 파라미터는 마치 요리의 '조미료'와 같다. 너무 많이 넣으면 맥락을 잃고, 너무 적게 넣으면 맛(개성)이 없다. 작업 목적에 맞는 최적의 조합을 제안한다.

작업 유형 권장 설정 (T / P / K) 기대 효과
논리적 추론 / 코드 작성 T: 0.1~0.3 P: 0.9 정답에 가까운 가장 확률 높은 단어만 선택하여 오류를 최소화함.
일반적인 대화 / 요약 T: 0.7 P: 0.95 적당한 유연함을 갖추면서도 대화의 맥락을 놓치지 않는 '표준' 설정.
창의적 글쓰기 / 시 / 소설 T: 0.9~1.2 P: 0.99 의외의 단어 선택을 허용하여 문학적 표현과 창의적인 전개를 유도함.
브레인스토밍 / 아이디어 T: 1.5 K: 100 확률이 낮은 기발한 단어들을 대거 포함시켜 파격적인 아이디어를 출력.
💡 Pro Tip: 온도(T)와 Top-P를 같이 쓸 때 보통 Temperature를 먼저 조절하여 전체적인 분위기를 정하고, Top-P로 '선 넘는 단어'들만 컷오프하는 방식이 가장 효과적이다. 예를 들어 T=1.0으로 창의성을 높였더라도 P=0.8을 주면, 확률 하위 20%의 무의미한 단어(오타나 외계어 등)가 섞이는 것을 방지할 수 있다.
SECTION 08

결론

실험하면서 알게 된 것

세 파라미터 모두 "값이 커질수록 확률이 분산되고, 작아질수록 한 단어에 집중된다"는 공통점이 있다. 그 점에서 세 파라미터 사이에 기능적인 상관관계가 있다고 볼 수 있다.

근데 작동하는 방식은 완전히 다르다. Temperature는 분포 모양 자체를 바꾸고, Top-P는 누적 확률로 후보를 동적으로 자르고, Top-K는 그냥 상위 K개만 남긴다. 실제 AI 시스템에서는 이 세 가지를 조합해서 쓰는데 서로 보완적인 역할을 한다.

가설 면에서는 Temperature랑 Top-K에 대한 생각이 대체로 맞았다. Top-P는 메커니즘 설명이 틀렸는데, "값이 클수록 다양성이 높아진다"는 방향은 맞았다.
